NPM

NPM (Node Package Manager) — это менеджер пакетов для JavaScript, позволяющий пользователям устанавливать, обновлять и управлять зависимостями в проектах.

Полезные команды

npm init - создаёт package.json

npm i {название пакета} - устанавливает пакет

npm i - устанавливает все зависимости


npm i redom@1.0.0 - устанавливает конкретную версию (сокращённый вариант)

npm install redom@1.0.0 - устанавливает конкретную версию (полный вариант)


npm un redom - удаляет пакет (сокращённый вариант)

npm uninstall redom - удаляет пакет (полный вариант)


npm up - обновляет все зависимости (сокращённый вариант)

npm update - обновляет все зависимости (полный вариант)


npm outdated - показывает устаревшие зависимости и какие есть обновления


npm i package@latest - устанавливает последнюю версию (сокращённый вариант)

npm install package@latest - устанавливает последнюю версию (полный вариант)

latest — это специальное слово, по которому команда npm понимает, что вы принудительно устанавливаете последнюю опубликованную стабильную версию пакета. Если не написать @latest, npm установит версию, которая удовлетворяет соответствующей записи в dependencies, но при обновлении может не подойти.

Запуск команд из package.json

npm run {название команды} - запускает команду

В качестве примера в package.json добавляем:

{
  "scripts": {
    "build": "parcel build index.html",
    "dev": "parcel index.html"
  }
}

npm run build - чтобы собрать проект

npm run dev - чтобы запустить проект(сервер)

Типы зависимостей

npm i -D parcel
npm i --save-dev parcel

Обе записи выполняют одно и то же действие. Помещают в секцию для разработки devDependencies, где dev — сокращение от development.


package.json после добавления пакетов:

{
  "dependencies": {
    "redom": "...",
    "navigo": "...",
    "v8n": "..."
  },
  "devDependencies": {
    "parcel": "..."
  }
}
